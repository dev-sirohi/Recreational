# 🧪 Recreational

A collection of small, focused projects built to explore different areas of programming, systems, and computer science.

This repository exists as a sandbox for experimentation — each project is an attempt to understand a concept deeply by implementing it from scratch rather than relying on abstractions.

---

## 🧠 Why this repo exists

I tend to get interested in a lot of different areas at once — systems, networking, compilers, functional programming, etc.

Instead of jumping randomly between topics, this repository helps me organize that curiosity into **contained, goal-oriented projects**. Each folder represents a deliberate attempt to learn something specific.

---

## 📦 Projects

### 🔹 Credis — In-Memory Database

**C# • TCP • Async • Networking**

A lightweight in-memory database implementing a byte-length-prefixed protocol (RESP-inspired). Focused on understanding networking, protocol design, and concurrent request handling.

→ `credis-a-c-sharp-in-memory-db`

---

### 🔹 Aligned Man Pages Library

**C# • CLI • Utilities**

A small utility library for rendering clean, aligned console output. Built to standardize formatting for CLI tools and improve readability.

→ `aligned-man-pages-library-in-c-sharp`

---

### 🔹 Huffman Encoding

**C • Data Structures • Algorithms**

Implementation of Huffman Encoding in C to explore compression, memory management, and low-level data structures.

→ `huffman-encoding-in-c`

---

### 🔹 Ngrok-style Tunneling (Learning Haskell)

**Haskell • Networking • Functional Programming**

An experimental project aimed at understanding tunneling and networking while learning Haskell and functional programming concepts.

→ `learn-haskell-project`

---

### 🔹 wc Command Reimplementation

**C# • CLI • Systems**

A reimplementation of the Linux `wc` command to explore file handling, performance considerations, and CLI design.

→ `linux-wc-command-reimplemented-in-c-sharp`

---

## 🧭 Philosophy

- Learn by building, not just reading
- Prefer low-level understanding over convenience
- Keep projects small, focused, and intentional
- Accept that not everything needs to be “production-ready”

---

## ⚠️ Disclaimer

This is a **learning repository**, not a production codebase.

You’ll find:

- inconsistent patterns between projects
- evolving coding styles
- experiments that may be incomplete or naive

That’s intentional — the goal here is growth, not perfection.
